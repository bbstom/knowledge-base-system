# 用户端提现记录显示修复

## 问题描述
管理员批准提现成功后，用户在佣金管理页面无法看到提现记录。

## 问题原因
Commission页面虽然调用了API获取提现记录，但没有在页面上显示提现记录的UI组件。页面只显示了"佣金记录"（commissionHistory），而没有显示"提现记录"（withdrawHistory）。

## 修复内容

### 1. 添加状态管理
```typescript
const [withdrawHistory, setWithdrawHistory] = useState<any[]>([]);
```

### 2. 保存提现记录数据
在 `loadCommissionData` 函数中添加：
```typescript
// 保存提现记录
setWithdrawHistory(withdrawOrders);
```

### 3. 添加提现记录显示UI
在"佣金记录"表格后面添加了完整的"提现记录"表格，包括：

**表格列**
- 订单号
- 金额
- 手续费
- 实际到账
- 状态（待审核/已完成/已拒绝）
- 时间

**状态标签**
- pending - 黄色标签（待审核）
- completed - 绿色标签（已完成）
- rejected - 红色标签（已拒绝）

**提示信息**
- 待审核：管理员正在审核您的提现申请
- 已完成：提现已成功，请查看您的钱包
- 已拒绝：提现被拒绝，佣金已退还到您的账户

## 修复后的效果

### 用户可以看到
1. ✅ 所有提现申请记录
2. ✅ 每条记录的详细信息（订单号、金额、手续费、实际到账）
3. ✅ 提现状态（待审核/已完成/已拒绝）
4. ✅ 申请时间
5. ✅ 友好的状态说明

### 数据流程
1. 用户申请提现 → 创建订单（pending状态）
2. 管理员审批 → 订单状态变为completed
3. 用户刷新页面 → 看到提现记录，状态显示"已完成"
